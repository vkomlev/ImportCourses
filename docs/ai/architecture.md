# Архитектура

## Модули

| Модуль | Файл | Ответственность |
|--------|------|------------------|
| `parser` | `src/parser.py` | HTTP-загрузка страниц (с ретраями), извлечение подкурсов уровня 1 (главная страница) и уровня 2 (страница подкурса), извлечение описания, извлечение материалов из секций «Текстовые уроки»/«Видеоуроки» |
| `uid_generator` | `src/uid_generator.py` | Транслитерация RU→EN, генерация slug, построение иерархического `course_uid`, обеспечение уникальности (числовой суффикс при коллизии) |
| `excel_writer` | `src/excel_writer.py` | Запись списка курсов в XLSX (лист `Courses`) и списка материалов в XLSX (лист `Materials`) через openpyxl |
| `main` | `src/main.py` | Оркестрация: обход подкурсов, привязка материалов к `course_uid`, генерация `external_uid` материалов, CLI (`argparse`), запись результатов |
| `config` | `src/config.py` | Все настраиваемые константы (см. [../configuration.md](../configuration.md)) |

Единственная внешняя зависимость времени выполнения — HTTP-доступ к сайту-источнику (`--url`); единственный побочный эффект — запись файлов в `output/`. Общей БД или бэкенда у парсера нет.

## Поток данных

```
--url (главная страница)
   │
   ▼
parser.parse_main_page()          → список Subcourse (уровень 1, с URL)
   │
   ▼  (для каждого подкурса уровня 1, один HTTP-запрос на страницу)
parser.parse_subcourse_page_with_materials()
   ├─ parser.extract_description()             → описание курса уровня 1
   ├─ parser._parse_subcourse_page_content()   → список Subcourse (уровень 2, из "Краткий план раздела")
   └─ parser.parse_materials_sections()        → список RawMaterial (текст, url, type)
   │
   ▼
main.parse_courses()
   ├─ uid_generator.generate_uid()              → course_uid для уровня 1 и уровня 2
   ├─ _material_title_matches_subcourse()       → привязка RawMaterial к course_uid (уровень 2 при совпадении текста, иначе уровень 1)
   └─ _unique_external_uid_for_course()         → external_uid материала (уникален в рамках course_uid)
   │
   ▼
excel_writer.write_courses_to_excel()   → output/courses_<timestamp>.xlsx (всегда, если есть курсы)
excel_writer.write_materials_to_excel() → output/materials_<timestamp>.xlsx (только если есть материалы)
```

## Алгоритм работы

1. **Парсинг главной страницы** — извлечение заголовков `<h3>` и ссылок «Перейти» под ними → подкурсы уровня 1.
2. **Генерация UID уровня 1** — для каждого подкурса первого уровня UID с префиксом главного курса (`MAIN_COURSE_UID`).
3. **Парсинг страниц подкурсов** — для каждого подкурса уровня 1 один HTTP-запрос к его странице:
   - описание (первый значимый параграф);
   - раздел «Краткий план раздела» → список подкурсов уровня 2;
   - секции «Текстовые уроки» и «Видеоуроки» → сбор ссылок (текст, URL, тип).
4. **Генерация UID уровня 2** — для каждого пункта плана UID с префиксом подкурса уровня 1.
5. **Привязка материалов** — для каждой ссылки: совпадение текста (полное или частичное, без учёта регистра) с названием подкурса уровня 2 → привязка к этому подкурсу; иначе → к подкурсу уровня 1 (родителю страницы). Генерация уникального `external_uid` в рамках `course_uid`.
6. **Запись в XLSX** — файл курсов (лист `Courses`) и, при наличии материалов, файл материалов (лист `Materials`).

## Генерация UID (`uid_generator`)

- Транслитерация русского текста в латиницу (`transliterate`, с ручным fallback-словарём, если библиотека не справилась).
- Приведение к slug: lowercase, пробелы/подчёркивания → дефис, удаление всех символов кроме `[a-z0-9-]`, схлопывание повторных дефисов.
- Префикс родительского UID через дефис (`{parent_uid}-{slug}`), обрезка по `UID_MAX_LENGTH`.
- Коллизии внутри одного запуска (`UIDGenerator._used_uids`) разрешаются числовым суффиксом (`-2`, `-3`, …); генератор одноразовый на запуск (`reset_uid_generator()` в начале `parse_courses`).

Пример: `"Основы Python"` → `osnovy-python`; с родителем `PY` → `PY-osnovy-python`.

## Границы доверия

- Вход — произвольный HTML с внешнего сайта; парсинг не выполняет код со страницы (BeautifulSoup, без исполнения `<script>`), но структура зависит от вёрстки сайта и может измениться без предупреждения (см. [../troubleshooting.md](../troubleshooting.md)).
- Выход — локальные XLSX-файлы в `output/`; дальнейшая загрузка в LMS выполняется вручную/отдельным процессом (Google Sheets + API импорта), не этим парсером.
