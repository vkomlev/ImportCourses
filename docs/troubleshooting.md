# Устранение проблем

## Не найдено ни одного курса (`sys.exit(1)` с "Не найдено ни одного курса!")

**Причина:** на главной странице не найдено тегов `<h3>` (см. `H3_TAG` в `src/config.py`) или под ними нет ссылки с текстом «Перейти» (`GO_LINK_TEXT`).

**Что делать:**
1. Откройте `--url` в браузере и проверьте, что структура страницы не изменилась (верстка сайта могла обновиться).
2. Запустите с `--verbose` — в логе будет видно, сколько заголовков `H3_TAG` найдено и почему ссылка «Перейти» не найдена для конкретного заголовка.
3. Если текст ссылки перехода на сайте изменился — обновите `GO_LINK_TEXT` в `src/config.py`.

## Подкурсы уровня 2 или материалы не собираются со страницы подкурса

**Причина:** не найден раздел «Краткий план раздела» (`PLAN_SECTION_HEADER`) или секции «Текстовые уроки» / «Видеоуроки» (`MATERIALS_HEADER_TEXT_LESSONS`, `MATERIALS_HEADER_VIDEO_LESSONS`).

**Что делать:** сверьте точный текст заголовков на странице подкурса с значениями в `src/config.py` (поиск — по частичному вхождению без учёта регистра, но заголовок должен присутствовать на странице). Подробнее о настройках — [configuration.md](configuration.md).

## Повторяющиеся сетевые ошибки при загрузке страниц

**Симптом:** в логе `Ошибка при загрузке {url}` несколько раз подряд, затем `Не удалось загрузить {url} после N попыток`.

**Причина:** сайт временно недоступен, либо `REQUEST_TIMEOUT` слишком мал для медленного соединения.

**Что делать:** парсер уже делает `RETRY_ATTEMPTS` попыток с нарастающей паузой (`RETRY_DELAY * попытка`). Если ошибка стабильно повторяется — проверьте доступность `--url` напрямую (curl/браузер) и увеличьте `REQUEST_TIMEOUT`/`RETRY_ATTEMPTS` в `src/config.py` при необходимости.

## Дубликаты UID в выходном файле

**Симптом:** ожидали один `course_uid`, а в файле встретился с суффиксом (`-2`, `-3`).

**Причина:** это штатное поведение — `uid_generator` добавляет числовой суффикс при совпадении slug у разных курсов (например, два подкурса с похожим названием). Не является ошибкой, но при импорте в LMS проверьте, что `parent_course_uid` для таких курсов различается корректно.

## Материал привязан не к тому подкурсу

**Причина:** привязка материала к `course_uid` работает по частичному текстовому совпадению названия ссылки с названием подкурса второго уровня (см. `_material_title_matches_subcourse` в `src/main.py`); при похожих названиях подкурсов возможна неоднозначность — побеждает первое совпадение по порядку подкурсов на странице.

**Что делать:** проверьте выходной XLSX материалов вручную перед импортом; при системных ошибках привязки — скорректируйте название подкурса на сайте или обработайте отдельно после экспорта.
